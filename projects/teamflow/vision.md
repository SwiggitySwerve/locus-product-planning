# TeamFlow - Vision

## What We're Building

TeamFlow is a **real-time collaborative project management platform** designed specifically for engineering teams. It combines the speed and simplicity of Linear with the flexibility and documentation capabilities of Notion.

## The Problem We're Solving

Engineering teams today face fragmented tooling:
- Project boards that don't sync in real-time
- Sprint planning disconnected from actual development work
- Time tracking that lives in separate apps
- No integration between code changes and task status
- Siloed information between PMs, engineers, and leadership

**Result**: Teams waste 20-30% of their time on tool-switching and status updates instead of building.

## Who It's For

### Primary Users

**Engineering Managers**
- Need: Sprint planning, velocity tracking, resource allocation
- Pain: Can't see real-time team progress without interrupting engineers
- Goal: Data-driven sprint decisions without micromanagement

**Software Developers**
- Need: Task management, time tracking, PR linking
- Pain: Updating tickets is disconnected from actual coding workflow
- Goal: Tasks that update automatically as they code

### Secondary Users

**Product Managers**
- Need: Roadmap visualization, stakeholder updates
- Pain: Building status reports manually
- Goal: Live dashboards they can share with stakeholders

**Executives**
- Need: High-level team performance metrics
- Pain: Waiting for weekly reports to understand delivery velocity
- Goal: Real-time visibility without micromanaging
